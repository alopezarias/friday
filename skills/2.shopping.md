# Skill: Lista de la compra

Gestiona la lista de la compra. Funciona tanto para planificación previa como para uso en tiempo real dentro del supermercado.

---

## Schema

```sql
CREATE TABLE IF NOT EXISTS lista_compra (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre TEXT NOT NULL,
    cantidad_necesaria REAL,
    cantidad_en_stock REAL DEFAULT 0,
    cantidad_a_comprar REAL,
    unidad TEXT,
    motivo TEXT,                        -- 'receta: Nombre receta' | 'objetivo nutricional' | 'manual'
    estado TEXT DEFAULT 'pendiente',    -- 'pendiente' | 'en_carrito' | 'comprado'
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Crea la tabla si no existe antes de cualquier operación de escritura.

## Generación automática de lista

Cuando se planifica un menú (desde la skill de entrenador) o el usuario pide generar la lista para un conjunto de recetas:

```python
import sqlite3
con = sqlite3.connect("data/friday.db")

# Para cada receta del menú:
for receta_id in recetas_del_menu:
    ingredientes = con.execute(
        "SELECT nombre, cantidad, unidad FROM receta_ingredientes WHERE receta_id = ?",
        (receta_id,)
    ).fetchall()

    for nombre, cantidad_necesaria, unidad in ingredientes:
        # Consultar stock actual
        row = con.execute(
            "SELECT SUM(cantidad) FROM alimentos WHERE LOWER(nombre) = LOWER(?)",
            (nombre,)
        ).fetchone()
        en_stock = row[0] or 0
        a_comprar = max(0, cantidad_necesaria - en_stock)

        if a_comprar > 0:
            # Si ya existe en lista con estado != 'comprado', sumar cantidad
            existente = con.execute(
                "SELECT id, cantidad_a_comprar FROM lista_compra "
                "WHERE LOWER(nombre) = LOWER(?) AND estado != 'comprado'",
                (nombre,)
            ).fetchone()
            if existente:
                con.execute(
                    "UPDATE lista_compra SET cantidad_a_comprar = ? WHERE id = ?",
                    (existente[1] + a_comprar, existente[0])
                )
            else:
                con.execute(
                    "INSERT INTO lista_compra (nombre, cantidad_necesaria, cantidad_en_stock, cantidad_a_comprar, unidad, motivo) "
                    "VALUES (?, ?, ?, ?, ?, ?)",
                    (nombre, cantidad_necesaria, en_stock, a_comprar, unidad, f"receta: {receta_nombre}")
                )

con.commit()
con.close()
```

## Estados y flujo en supermercado

| estado | significado | cómo se activa |
|--------|-------------|----------------|
| `pendiente` | está en la lista, no se ha cogido | estado inicial |
| `en_carrito` | ya cogido del lineal, en el carrito | "he cogido X", "X al carrito", "tengo el X" |
| `comprado` | ya pagado y en casa | "ya he pagado", "comprado todo", "listo" |

**Flujo ítem a ítem:**
- "He cogido el pollo" → `UPDATE lista_compra SET estado='en_carrito' WHERE nombre LIKE '%pollo%' AND estado='pendiente'`
- "Ya tengo todo en el carrito" → marcar todos los `pendiente` como `en_carrito`
- "He pagado" / "listo, ya estoy fuera" → marcar todos los `en_carrito` como `comprado`

**Tras marcar como comprado:**
Preguntar: "¿Actualizo el stock con lo que has comprado?" Si confirma → hacer upsert en `alimentos` con las cantidades compradas.

## Adición manual

El usuario puede añadir ítems que no vengan de recetas:
- "Añade papel de cocina a la lista" → insertar con `motivo='manual'`
- "Necesito comprar 2 litros de leche" → insertar con cantidad y unidad

---

## Consultas

Al preguntar por la lista:
- Mostrar agrupado por estado: primero `pendiente`, luego `en_carrito`
- Indicar el motivo (de qué receta viene cada ítem)
- Incluir cantidad y unidad

Al preguntar "¿qué recetas podría hacer si compro todo lo pendiente?":
- Cruzar `lista_compra.nombre` + `alimentos` actuales contra `receta_ingredientes`
- Listar las recetas que quedarían completas

---

## Limpieza

Cuando el usuario diga "limpia la lista" o "borra lo comprado":
- Eliminar todos los ítems con `estado='comprado'`
- Confirmar cuántos ítems se han eliminado
