# Skill: Recetas

Gestiona el recetario personal. Propone recetas, las guarda con confirmación explícita del usuario, calcula sus macros y mantiene siempre una foto asociada.

---

## Schemas

```sql
CREATE TABLE IF NOT EXISTS recetas (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre TEXT NOT NULL UNIQUE,
    descripcion TEXT,
    instrucciones TEXT,
    porciones INTEGER DEFAULT 1,
    tiempo_prep_min INTEGER,
    calorias_porcion REAL,
    proteinas_porcion REAL,
    carbohidratos_porcion REAL,
    grasas_porcion REAL,
    foto_path TEXT,                 -- relativo al proyecto: data/photos/recipes/uuid.jpg
    fuente TEXT DEFAULT 'llm',      -- 'usuario' | 'llm'
    presupuesto_estimado REAL,
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS receta_ingredientes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    receta_id INTEGER NOT NULL REFERENCES recetas(id) ON DELETE CASCADE,
    nombre TEXT NOT NULL,           -- usar mismo nombre que alimentos.nombre cuando sea posible
    cantidad REAL NOT NULL,
    unidad TEXT NOT NULL
);
```

Crea ambas tablas si no existen antes de cualquier operación de escritura.

---

## Flujo de guardado

**Solo se guarda cuando:**
1. El usuario lo pide explícitamente: "guarda esta receta", "añade esto al recetario"
2. El LLM propone guardar y el usuario confirma con "sí", "dale", "guárdala" o equivalente

Nunca guardar sin confirmación. Si el LLM propone una receta, al final pregunta: "¿Guardo esta receta en el recetario?"

---

## Propuesta de recetas

Cuando el usuario pida sugerencias o el sistema necesite opciones para un menú:

1. Consultar `alimentos` — qué hay disponible y en qué cantidades
2. Consultar `recetas` existentes — qué hay ya guardado
3. Proponer recetas priorizando el uso del stock existente
4. Si el usuario indica presupuesto, incluir también recetas con ingredientes a comprar dentro de ese límite
5. Para cada propuesta mostrar:
   - Nombre y descripción breve
   - Ingredientes marcados: ✓ en casa / ✗ hay que comprar
   - Macros estimadas por porción
   - Tiempo de preparación estimado
   - Presupuesto estimado si requiere compras

## Foto del plato

Toda receta guardada debe tener foto. No hay excepciones.

**Caso 1 — El usuario envía una foto:**
El path de la imagen llegará en el input como `data/photos/recipes/uuid.jpg`. Guardar ese path en `recetas.foto_path`.

**Caso 2 — El LLM propone la receta (sin foto del usuario):**
Buscar una imagen representativa en internet y descargarla:

```python
import requests, uuid
from pathlib import Path

def descargar_foto_receta(nombre_receta: str) -> str | None:
    # Buscar imagen usando DuckDuckGo images o similar sin API key
    query = nombre_receta.replace(" ", "+") + "+plato+receta"
    try:
        # Intentar obtener URL de imagen de búsqueda
        headers = {"User-Agent": "Mozilla/5.0"}
        r = requests.get(f"https://duckduckgo.com/?q={query}&iax=images&ia=images", headers=headers, timeout=5)
        # Extraer primera URL de imagen del HTML (patrón básico)
        import re
        urls = re.findall(r'"thumbnail":"(https://[^"]+)"', r.text)
        if not urls:
            return None
        img_url = urls[0]
        img_data = requests.get(img_url, headers=headers, timeout=5).content
        path = Path(f"data/photos/recipes/{uuid.uuid4().hex}.jpg")
        path.parent.mkdir(parents=True, exist_ok=True)
        path.write_bytes(img_data)
        return str(path)
    except Exception:
        return None
```

Si la descarga falla, guardar la receta con `foto_path = NULL` e informar al usuario: "Receta guardada. No pude obtener una foto automáticamente — envíame una cuando puedas."

**Actualización de foto:**
Si el usuario envía una foto con "actualiza la foto de [nombre]", reemplazar `foto_path` de esa receta con el nuevo path.

## Recuperación de recetas

Al pedir una receta (por nombre, ingrediente, objetivo nutricional o para un menú):

Formato de respuesta:
```
[Nombre de la receta]
[Descripción breve]

Ingredientes ([porciones] porciones):
  - Xg ingrediente1  ✓
  - Xml ingrediente2  ✗ (hay que comprar)

Preparación:
  1. Paso uno
  2. Paso dos...

Macros por porción:
  Calorías: X kcal | Proteína: Xg | Carbos: Xg | Grasas: Xg

Tiempo: X min | Presupuesto est.: X€
```

Indicar siempre si faltan ingredientes en el stock actual.

---

## Consistencia de nombres

Usar el mismo nombre de ingrediente que en `alimentos`. Si el ingrediente de la receta no coincide exactamente con un nombre en `alimentos`, intentar el match con `LOWER()` o por similitud. Si no hay coincidencia, registrar el nombre tal cual y anotar que no se pudo cruzar con el stock.
