# Skill: Seguimiento de progreso

Historifica el peso y las medidas corporales para ver evolución real hacia el objetivo. El entrenador actualiza el perfil con el peso actual; esta skill guarda el historial y calcula tendencias.

---

## Schemas

```sql
CREATE TABLE IF NOT EXISTS historial_peso (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fecha TEXT NOT NULL UNIQUE,       -- 'YYYY-MM-DD'
    peso_kg REAL NOT NULL,
    notas TEXT
);

CREATE TABLE IF NOT EXISTS medidas_corporales (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fecha TEXT NOT NULL,              -- 'YYYY-MM-DD'
    cintura_cm REAL,
    cadera_cm REAL,
    pecho_cm REAL,
    brazo_cm REAL,
    muslo_cm REAL,
    notas TEXT
);
```

---

## Registro de peso

Cuando el usuario diga "peso X kilos", "me he pesado, X kg", o actualice su peso en el perfil del entrenador:

1. Actualizar `perfil_usuario.peso_kg`
2. Insertar en `historial_peso` si no hay entrada para hoy

```python
import sqlite3
from datetime import date
```

## Registro de medidas

Cuando el usuario dé medidas corporales: guardar en `medidas_corporales`. No todas son obligatorias — guardar las que proporcione y dejar el resto en NULL.
