# Skill: Suplementos y medicación

Gestiona suplementos deportivos y medicación diaria. Registra qué hay que tomar, cuándo, y si se ha tomado.

---

## Schemas

```sql
CREATE TABLE IF NOT EXISTS suplementos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nombre TEXT NOT NULL UNIQUE,
    dosis TEXT NOT NULL,              -- '5g', '1 cápsula', '2 pastillas'
    frecuencia TEXT NOT NULL,         -- 'diario' | 'dias_entrenamiento' | 'semanal'
    momento TEXT,                     -- 'mañana' | 'antes_entreno' | 'con_comida' | 'antes_dormir'
    notas TEXT,
    activo INTEGER DEFAULT 1,
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS toma_suplementos (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    suplemento_id INTEGER NOT NULL REFERENCES suplementos(id),
    fecha TEXT NOT NULL,              -- 'YYYY-MM-DD'
    tomado INTEGER DEFAULT 0,
    hora TEXT,                        -- 'HH:MM', opcional
    UNIQUE(suplemento_id, fecha)
);
```

---

## Alta de suplemento

Cuando el usuario mencione que toma algo (creatina, proteína, vitaminas, medicación):

Preguntar:
1. Dosis (ej: "5 gramos", "1 cápsula")
2. Frecuencia: todos los días, solo días de entrenamiento, o semanal
3. Momento del día (mañana, antes de entrenar, con la comida, antes de dormir)

Guardar y confirmar. No preguntar todo a la vez si el usuario ya dio parte de la info.

## Consultas

**Resumen diario:**
```python
hoy = date.today().isoformat()
pendientes = con.execute("""
    SELECT s.nombre, s.dosis, s.momento
    FROM suplementos s
    LEFT JOIN toma_suplementos t ON s.id = t.suplemento_id AND t.fecha = ?
    WHERE s.activo = 1 AND s.frecuencia = 'diario'
    AND (t.tomado IS NULL OR t.tomado = 0)
""", (hoy,)).fetchall()
```

Responder cuando el usuario pregunte:
- "¿Me he tomado todo hoy?" → revisar tomas del día
- "¿Qué me falta tomar?" → suplementos activos sin toma hoy
- "¿Cuántos días llevo con la creatina?" → contar tomas de los últimos 30 días
