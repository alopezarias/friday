# Skill: Diario alimentario

Registra lo que realmente se come cada día, no lo que el plan dice. Compara con los objetivos del entrenador y calcula la adherencia real.

---

## Schema

```sql
CREATE TABLE IF NOT EXISTS diario_alimentario (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fecha TEXT NOT NULL,                  -- 'YYYY-MM-DD'
    tipo_comida TEXT NOT NULL,            -- 'desayuno' | 'almuerzo' | 'comida' | 'merienda' | 'cena' | 'extra'
    descripcion TEXT NOT NULL,            -- lo que se comió, en lenguaje libre o nombre de receta
    receta_id INTEGER REFERENCES recetas(id),  -- si corresponde a una receta guardada
    calorias REAL,
    proteinas REAL,
    carbohidratos REAL,
    grasas REAL,
    segun_plan INTEGER DEFAULT 0,         -- 1 si coincide con menu_diario
    notas TEXT,
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

## Registro de una comida

Cuando el usuario diga "he comido X", "me he tomado Y", "cené Z":

1. Identificar fecha y tipo de comida (si no lo dice, asumir la comida más cercana a la hora actual)
2. Si corresponde a una receta guardada → vincular con `receta_id` y copiar sus macros
3. Si no es una receta → estimar macros con conocimiento del modelo
4. Verificar si coincide con lo planificado en `menu_diario` para esa fecha y tipo de comida → marcar `segun_plan`
5. Guardar en `diario_alimentario`
6. Confirmar brevemente: "Anotado. +Xcal, +Xg proteína."

```python
import sqlite3
from datetime import date

con = sqlite3.connect("data/friday.db")
con.execute("""
    INSERT INTO diario_alimentario
    (fecha, tipo_comida, descripcion, receta_id, calorias, proteinas, carbohidratos, grasas, segun_plan)
    VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
""", (fecha, tipo_comida, descripcion, receta_id, calorias, proteinas, carbohidratos, grasas, segun_plan))
con.commit()
con.close()
```

## Balance del día

Cuando el usuario pregunte "¿cómo voy hoy?" o "¿cuántas calorías llevo?":

```python
hoy = date.today().isoformat()
totales = con.execute("""
    SELECT
        SUM(calorias) as kcal,
        SUM(proteinas) as prot,
        SUM(carbohidratos) as carbs,
        SUM(grasas) as grasas
    FROM diario_alimentario WHERE fecha = ?
""", (hoy,)).fetchone()

# Comparar con perfil_usuario.calorias_objetivo, proteinas_g, etc.
```

Mostrar:
```
Hoy [fecha]:
  Consumido:  Xkcal / Xp / Xc / Xg
  Objetivo:   Xkcal / Xp / Xc / Xg
  Diferencia: ±Xkcal  (±Xp proteína)
```

Si faltan más de 30g de proteína al final del día, mencionarlo.

## Adherencia semanal

```python
# Calcular % de comidas que coincidieron con el plan
semana_inicio = ...  # lunes de la semana actual
total_planificadas = con.execute(
    "SELECT COUNT(*) FROM menu_diario WHERE fecha >= ? AND fecha <= ?",
    (semana_inicio, hoy)
).fetchone()[0]

seguidas = con.execute(
    "SELECT COUNT(*) FROM diario_alimentario WHERE fecha >= ? AND segun_plan = 1",
    (semana_inicio,)
).fetchone()[0]

adherencia = (seguidas / total_planificadas * 100) if total_planificadas else 0
```

Responder a:
- "¿Cuánto he cumplido el plan esta semana?" → porcentaje de adherencia
- "¿Qué he comido hoy?" → listar entradas del día
- "¿Cuántas calorías he comido esta semana?" → suma de la semana
- "¿Estoy cumpliendo mis macros?" → comparar acumulado vs. objetivo diario × días transcurridos

## Detección de patrones

Si el usuario lleva 3+ días de datos, detectar y mencionar (máximo una vez por semana):
- Comida que se salta habitualmente ("sueles no registrar merienda")
- Días de la semana con menor adherencia
- Si sistemáticamente se queda por debajo de proteína

No ser repetitivo — si ya se mencionó un patrón esta semana, no volver a mencionarlo.
